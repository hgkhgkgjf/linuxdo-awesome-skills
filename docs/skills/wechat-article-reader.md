# wechat-article-reader

## 快速判断

`wechat-article-reader` 是一个轻量级微信公众号文章读取 skill。它通过 `curl` 模拟浏览器请求拿到文章 HTML，再用 Python 正则从 `js_content` 区域提取正文，适合快速读取、摘要、保存或提取图片链接。

## 适合场景

- 你收到一个 `mp.weixin.qq.com/s/...` 文章链接，想快速知道文章内容。
- 你希望把微信公众号文章保存为纯文本，方便后续阅读、归档或分析。
- 你只需要摘要、全文、图片链接或原始 HTML，不想启动浏览器自动化。
- 你要批量处理多个公众号文章链接。
- 你希望用轻量脚本辅助理解别人分享的微信文章。

## 前置要求

- `curl` 命令行工具。
- Python 3.6+。
- 可访问的微信公众号文章链接。
- 网络可以访问 `mp.weixin.qq.com`。
- 如需批量保存，需要准备输出目录或文件命名规则。

## 输入与输出

输入：

- 微信公众号文章 URL
- 可选输出文件路径
- 可选模式：摘要、全文、图片链接、原始 HTML、调试日志

输出：

- 文章标题
- 文章摘要
- 完整正文文本
- 图片链接列表
- 原始 HTML 文件
- 调试信息

## 核心特色

- 轻量：只依赖 `curl` 和 Python 标准库，不需要浏览器或 Playwright。
- 快速：项目 README 标注单篇获取通常在 3 秒内完成。
- 直接：从 HTML 源码里的 `js_content` 区域提取正文。
- 多输出：支持摘要、全文、图片链接和原始 HTML。
- 易调试：可以保存原始 HTML，检查 `js_content` 是否存在。
- 可批量：适合把多个公众号链接循环处理为本地文本。

## 和相近技能的差异

- 相比 [karpathy-llm-wiki](karpathy-llm-wiki.md)，`wechat-article-reader` 不维护长期知识库，只负责读取和提取微信公众号文章。
- 相比 [graphify](graphify.md)，它不构建 Neo4j 图谱，也不做 GraphRAG，而是把单篇或多篇公众号文章变成可读文本和资源链接。
- 相比浏览器自动化方案，它更轻更快，但不负责完整渲染复杂交互内容。
- 相比当前多个 PPT 类技能，它不生成演示稿，核心产物是文章正文、摘要、图片链接或原始 HTML。

## 工作流程

1. 准备微信公众号文章链接。
2. 脚本使用 `curl` 携带浏览器 User-Agent、Referer 等请求头获取 HTML。
3. Python 正则匹配 `id="js_content"` 对应正文区域。
4. 清理 HTML 标签、换行、空白字符和编码问题。
5. 按参数输出摘要、全文、图片链接或原始 HTML。
6. 如果提取失败，使用 `--verbose` 或 `--html` 保存调试文件。
7. 批量场景下循环处理 URL 列表并写入本地文件。

## 当前限制

- 微信页面结构或反爬策略变化可能导致提取失败。
- 只适合可访问的微信公众号文章链接，不处理需要特殊登录态或私密转发的内容。
- 正文提取依赖 `js_content` 结构，复杂排版、视频、小程序卡片等内容可能丢失。
- 使用提取内容时应尊重原作者版权和平台规则。
- 它是轻量读取工具，不提供浏览器渲染、OCR 或文章语义校对能力。

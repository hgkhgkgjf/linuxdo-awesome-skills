# guizang-ppt-skill

## 快速判断

`guizang-ppt-skill` 是一个 HTML 演示稿生成工作流。它把文章、Markdown、大纲或素材整理成单文件横向翻页 deck，并用电子杂志风或瑞士国际主义风完成排版、配图、封面和浏览器演示。

## 适合场景

- 你想把长文章、技术分享、方法论或产品发布内容整理成 6 到 10 页左右的演讲稿。
- 你希望演示稿更像设计过的网页 deck，而不是模板味很重的传统 PPT。
- 你需要电子杂志风、瑞士风、强标题、数据大字报、图片网格、流程图或对比页。
- 你还想顺手生成公众号头图、分享卡、小红书封面、视频号横版封面等视觉素材。
- 你愿意用浏览器直接演示单文件 HTML，或把它作为后续转制、截图、录屏的视觉底稿。

## 输入与输出

输入：

- 文章、Markdown、旧 PPT、产品文档、分享主题或演讲大纲
- 可选图片、截图、数据、品牌约束和封面规格

输出：

- 一份单文件 HTML 横向翻页演示稿
- 电子杂志风或瑞士国际主义风的完整视觉排版
- 可选 PPT 配图、信息图、截图风格化素材
- 可选公众号 `21:9` 头图、`1:1` 分享卡、小红书 `3:4` 封面、视频号横版封面
- 浏览器预览、自检清单和瑞士风版式校验结果

## 核心特色

- 单文件 HTML 交付：不需要构建或服务器，浏览器直接打开即可演示。
- 双视觉系统：电子杂志风偏叙事、观点和个人表达；瑞士国际主义风偏事实、产品、分析和方法论表达。
- 横向翻页体验：支持键盘、滚轮、触屏、底部圆点和索引视图等演示交互。
- 版式约束清晰：电子杂志风提供 10 种布局骨架；瑞士风提供 22 种锁定版式和校验器。
- 主题色收敛：用预设主题保护整体审美，避免每页自由发挥导致风格漂移。
- 视觉素材链路完整：同一套规则可覆盖演示稿、配图、截图再设计和多平台封面。
- 低性能静态模式：需要时可关闭 WebGL / canvas 动效，让内容回到稳定可读状态。

## 工作流程

1. 澄清风格、受众、时长、素材、图片需求、主题色和硬约束。
2. 在电子杂志风和瑞士国际主义风之间选择一种视觉系统。
3. 拷贝对应 HTML 模板并替换标题等必填占位符。
4. 从预设主题色中选择一套，不随意自定义颜色。
5. 先规划主题节奏和页码结构，再从布局骨架中选页面。
6. 填充文案、数据、图片和说明，必要时生成配图或截图风格化素材。
7. 对照 checklist 做视觉和代码自检；瑞士风可运行版式校验器。
8. 用浏览器打开预览，逐页检查字号、间距、图片落位、动效和底部分页安全区。

## 当前限制

- 更适合 6 到 10 页左右的观点型或发布型演示；页数太多时风格可能显得重复。
- 核心交付是 HTML deck，不是原生 PPTX；多人协作编辑和 PPTX 导出不是主流程。
- 不适合大段表格数据、超高信息密度培训课件或需要复杂图表堆叠的材料。
- 输出质量依赖前期风格选择、素材质量、布局匹配和浏览器预览复核。

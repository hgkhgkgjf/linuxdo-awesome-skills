# image-to-code-skill

## 快速判断

`image-to-code-skill` 是一个图片转代码工作流。它把 UI 图片、截图或 Figma 导出稿还原为自适应前端代码，同时导出高清透明 PNG 切图资源，并可继续拆分为可编辑 Figma 图层。

## 适合场景

- 你拿到一张 AI 生成的手机端页面图，想尽快还原成 HTML 页面。
- 你已有 UI 设计图、截图或 Figma 导出稿，希望严格按原图复刻。
- 你不想只得到“看起来差不多”的页面，而是需要可复核的坐标、图层、切图和 QA 流程。
- 你希望最后还能导入 Figma 编辑，而不是得到一份纯截图式页面。

## 输入与输出

输入：

- 一张 UI 图片、截图、App/Web 设计稿或 Figma 图片导出稿
- 截图里的文字、图标、插画、按钮、背景等界面元素

输出：

- 一份基于 `750px` 锁定画板的自适应前端代码
- 一套 2x/3x/4x 高清透明 PNG 切图资源
- 一份 `layers.manifest.json`，记录图层 bbox、z-index、资源路径、PNG 像素尺寸和 CSS 显示尺寸
- bbox 预览、PNG 透明度审计、截图对比等 QA 产物
- 可选 Figma 图层规格或本地 importer

## 核心特色

- 原图就是视觉合同：不重新设计、不主观优化间距、不用相似图标或占位素材替代。
- `750px` 是测量、定位和 QA 的基准坐标系，最终页面再通过外层 fit wrapper 按容器或手机视口等比缩放。
- `layers.manifest.json` 是布局、切图、代码、QA 和 Figma 导出的共同数据源。
- PNG 资源强调高清文件、固定显示尺寸、透明背景和防裁切，默认至少 2x，小图标或低清源图可用 3x/4x。
- 文本和简单图形尽量代码化或 Figma 原生 shape，复杂图标、插画、头像和产品图使用独立 PNG。
- 交付前要求做 750px 基准截图复核、375px 和至少一个其他手机宽度的自适应检查。

## 工作流程

1. 检查截图中的元素、文字和尺寸。
2. 将原图归一化到 `750px` 的统一手机坐标体系。
3. 生成 `layers.manifest.json`。
4. 切图前预览每个目标资源的 bbox。
5. 将资源按 `2x` 到 `4x` 放大导出，提升清晰度。
6. 文本和简单图形使用代码实现，图标和插画切图并尽量使用透明背景。
7. 验证 `375px` 以及其他手机宽度下的适配情况。
8. 通过截图和复核步骤检查最终结果。
9. 可选输出 Figma 图层规格，或配合相关 web-to-figma 插件使用。

## 当前限制

- 输出质量依赖源图清晰度、bbox 测量、manifest 完整度和透明 PNG 审计。
- Figma 输出通常需要额外生成 layer spec 或本地 importer；只有具备 Figma 写入权限时才适合直接写入远端文件。
- 当前重点是严格视觉还原，不适合作为“重新设计”或“自动优化界面”的工具。

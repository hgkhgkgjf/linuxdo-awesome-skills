# awesome-ppt-skills

## 快速判断

`awesome-ppt-skills` 是一个图片优先的 PPTX 生成工作流。它把一句提示词、简报或源文件整理成演示稿结构，再让图像模型把每页的标题、正文、标签、图示和视觉设计整体渲染成整页图片，最后组装成 PPTX。

## 适合场景

- 你想从一个主题快速生成一份视觉完整的 PPTX。
- 你希望每页像完整设计稿，而不是只有大纲、背景图或普通文本框。
- 你需要技术分享、产品发布、主题演讲、概念说明等偏视觉表达的演示稿。
- 你可以接受稳定版先输出图片优先 PPTX，文字主要烘焙在整页图片中。
- 你愿意把可编辑版本作为实验性重建结果，而不是无损转换承诺。

## 输入与输出

输入：

- 演示主题、提示词、简报、文章或源文件
- 可选页数、比例、语言、视觉风格和输出路径

输出：

- 一份图片优先的 PPTX，每页通常是一张完整渲染的幻灯片图片
- 一份 `deck.json`，记录页数、文案、prompt、图片路径和构建信息
- 每页幻灯片图片
- 可选 `ppt-master` handoff 和实验性可编辑 PPTX

## 核心特色

- 整页成图：不是只生成背景，而是把标题、正文、标签、图示和设计一起生成到每页图片里。
- 风格先行：先识别主题和受众，再从主题风格库中选择并改写视觉方向。
- 文本进入 prompt：每页可见文字会写入 `rendered_text` 和图片 prompt，便于生成与后续重建。
- 稳定与实验分离：`awesome-ppt-std` 专注稳定图片版 PPTX；`awesome-ppt-editable` 才进入可编辑重建流程。
- 可复现：`deck.json` 保存 slide plan、文案、prompt 和图片路径，便于重新生成或迭代。
- 可编辑重建链路：实验流程会把图片版 deck 交给 `ppt-master`，尝试重建原生文本、形状、图表和对象。

## 工作流程

1. 解析提示词、页数、比例、语言、风格和输出路径。
2. 检查源文件，提取主题、证据、引用、数据、图片和限制条件。
3. 选择适合主题的视觉风格，并调整为当前受众和内容服务。
4. 创建 deck brief、叙事结构和每页 slide spec。
5. 为每页定义完整 `rendered_text`，并写入对应图片生成 prompt。
6. 生成每页整图，检查标题、正文、标签和图示文字是否准确。
7. 校验 `deck.json`、图片路径、页数和比例。
8. 组装成图片优先 PPTX。
9. 如选择可编辑流程，再导出 `ppt-master` handoff 并重建可编辑 PPTX。
10. 检查 PPTX 页数、媒体数量、原生文本、必要文字和明显颜色异常。

## 当前限制

- 标准流程的可见文字默认烘焙在整页图片中，不是原生可编辑文本。
- 可编辑流程是基于图片参考和 `rendered_text` 的重建，不是无损位图转矢量。
- 密集表格、精确合规文本、财务或法律类逐字审计内容不适合默认图片优先流程。
- 输出质量依赖图像模型文字渲染能力、prompt 质量、逐页检查和必要重生成。

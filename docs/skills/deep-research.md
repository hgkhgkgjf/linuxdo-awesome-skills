# deep-research

## 快速判断

`deep-research` 是一个深度调研报告生成 skill。它不是简单的“搜索后总结”，而是围绕一个主题自动完成大纲规划、资料采集、本地资料读取、并行章节撰写、引用转换和质量检查，最终输出带目录、元数据、参考来源和免责声明的 Markdown 调研报告。

## 适合场景

- 你要围绕行业、市场、竞品、政策、技术或社区主题做公开资料调研。
- 你希望从一个主题快速生成结构完整、可继续修改的研究报告。
- 你需要多语言输出，且希望系统按目标语言搜索和撰写，而不是只做翻译。
- 你有本地 MD、TXT、PDF 或 DOCX 资料，希望整理为标准报告。
- 你更重视来源可追溯、正反观点并存和数据口径检查，而不是短摘要。

## 前置要求

- LLM 运行时或可适配的 Agent 工具。
- 在线模式需要 Scrapling 与网络访问。
- 默认使用内置 SearXNG 搜索端点。
- 本地资料模式支持 MD、TXT、PDF、DOCX。
- 非 OpenCode 环境需要适配命令入口、多 Agent 编排和更新方式。

## 输入与输出

输入：

- `/research 主题`
- 可选深度模式：quick、standard、deep
- 可选本地资料目录或文件
- 可选联网补充或只看本地资料的约束

输出：

- Markdown 深度调研报告
- 报告目录、元数据、阅读时间和数据截至日期
- 章节正文、数据表、正反观点和结论判断
- 文内引用与参考来源列表
- 免责声明和质量检查后的最终装配结果

## 核心特色

- 四阶段流水线：分析大纲、采集数据、并行撰写、装配验收。
- 多语言输出：自动检测主题语言，按目标语言进行资料搜索与报告撰写。
- 三档深度：quick、standard、deep 对应不同章节数、段落数和篇幅目标。
- 在线与离线兼容：可联网搜索公开资料，也可只读取本地文件生成报告。
- 多层搜索链路：大纲建议源、SearXNG、内置 sources.json 和免费源补强。
- 质量门：强调结论先行、来源可追溯、反方观点、三层深度、目录和元数据完整。

## 和相近技能的差异

- 相比 [graphify](graphify.md)，`deep-research` 不构建 Neo4j 知识图谱，核心产物是可阅读、可归档、可继续编辑的 Markdown 深度报告。
- 相比 [karpathy-llm-wiki](karpathy-llm-wiki.md)，它不是长期维护某个主题 wiki，而是围绕具体调研主题自动生成报告。
- 相比 [wechat-article-reader](wechat-article-reader.md)，它不只是提取单篇文章正文，而是做调研规划、资料采集、章节撰写和质量验收。
- 相比 PPT 类技能，它不直接生成演示稿，但生成的研究报告可以作为后续 PPT 生成技能的输入材料。

## 工作流程

1. 输入 `/research 主题`，或选择 quick、standard、deep 等深度模式。
2. 自动识别主题语言，确定报告语言和调研目标。
3. 生成调研大纲、章节结构和搜索计划。
4. 在线模式并行使用建议源、SearXNG、内置 sources.json 和免费源补强，再用 Scrapling 批量抓取全文。
5. 离线模式读取用户指定的 MD、TXT、PDF 或 DOCX 本地资料。
6. 把资料整理为数据池，并按章节并行撰写正文。
7. 执行 validate、assemble-report、convert-citations、qa-report 等验收流程。
8. 输出带目录、元数据、引用、参考来源和免责声明的 Markdown 报告。

## 当前限制

- 在线模式需要 Scrapling、网络访问和可用搜索链路，部署成本高于轻量脚本型 skill。
- 非 OpenCode 环境需要把命令入口、多 Agent 编排和更新机制适配到当前工具。
- 报告生成时间和 token 消耗会随主题复杂度、深度模式和资料数量增加。
- 它主要基于公开资料和用户提供文件，不等同于付费数据库或一手调研。
- 长报告仍需人工复核事实、引用、数据口径和结论适用范围。
